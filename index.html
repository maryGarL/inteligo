EL INICIO

Este proyecto se desarrollo dentro de el Talent Fest-Lima organizado por Laboratoria , Utilizamos para ello las metodologías de User Experience Design: Design Thinking, Lean UX y Design Sprint, todas ellas metodologías de diseño de experiencia de uruario para este propósito. Luego se utilizo lenguaje de programación JavaScript para la creación del algoritmo y la creación y conexión de API con la base de datos.

Flujo de Procesos

Todo el equipo se reunió conjuntamente con los stakeholder para definir y aterrizar el proyecto y definir su alcance mediante un flujo de procesos que permite identificar detelladamente cada paso dentro de la experiencia en la plataforma.

LISTA DE REQUERIMIENTOS

Las características que debia tener la plataforma son parte de la información obtenida de las entrevistas con los stakeholders, de ellos podemos destacar algunos requerimientos como:

  • Desarrollar vista para movil
  • Desarrollar vista para Escritorio
  • Desarrollar vista para tabletas
  • Respetar el lenguaje de marca
  • Generar contenido visualmente amigable
  • Mejorar el tamaño de la tipografía.
  • Generar un reporte de estado de perfil en formato PDF.
  • Generar servicios de mailing despues del perfilamiento

PRIORIZACIÓN

Fue de las partes mas difíciles a realizar ya que la mayoria de los requerimiento eran realmente indispensables para la presenntación de un MVP (Minimun Viable Product), Pero asumiendo la responsabilidad de entregar un producto en menor tiempo, se tomo la decision de priorizar las siguientes carcteristicas en el proyecto:

  • Desarrollar vista para Escritorio
  • Respetar el lenguaje de marca
  • Generar contenido visualmente amigable
  • Mejorar el tamaño de la tipografía.
  • Generar un reporte de estado de perfil en formato PDF.

PROTOTIPADO

El prototipado consistió en realizar dibujos de la pantallas con sus diferentes caracteristicas en papel o mediante programas de diseño. Para el protipado se priorizó Desktop First ya que la plataforma en su primera version esta orientada a un emulador de escritorio en tablet.

Para este proyecto realizamos ambos tipos de prototipados:

PROTOTIPADO EN PAPEL

Nos brindó la ventaja de alterar cualquiLer carcterística que se desee mejorar o suprimir de forma más rápida pero en baja calidad,además de tener la posibilidad de hacer testeos en menor tiempo.

PROTOTIPADO DIGITAL

Este fue realizado con ayuda de Programas de diseño y Edición como Marvel, Scketch e Ilustrator, se obtuvo un demo en alta fidelidad respetando el lenguaje de marca que ayudo mejorar la experiencia de testeo y generar opiniones mas realistas acercas del proyecto.

TESTING

Fue un proceso de retroalimentación constante, nos permitió identificar que era lo que funcionaba mejor dentro de la plataforma y que es lo que debiamos mejorar. La frase "No te enamores de lo que diseñas" nunca fue más precisa. Se utilizó la plataforma de Marvel para montar el prototipo y crear los enlaces necesarios para poder darle un enfoque funcional

A/B Testing

Consiste en realizar la prueba de 2 diseños diferentes para una misma carcterística funcional y en base al resultado que se obtenga de cada uno, elegir cual es el más eficiente y de mejor deempeño dentro de la aplicación.

Se realizaron cinco testeos de la aplicación, los dos primeros fueron realizados con analístas de INTELIGO SAB y los otros 3 fueros realizados con clientes prospectos que pudimos encontrar dentro del Talent Fest
